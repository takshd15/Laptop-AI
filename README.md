# Laptop AI

Laptop AI is a local-first command-line assistant for your notes. It indexes a
folder, stores chunks in a custom vector database, and answers questions using
local Ollama models.

The v1 flow is:

```sh
laptop-ai init
laptop-ai index ./examples/notes
laptop-ai ask "what did I write about cats?"
```

It returns an answer plus source files, without sending your notes to a cloud
model.

## What Is Laptop AI?

Laptop AI turns local files into searchable memory. It scans supported files,
splits them into chunks, embeds each chunk with a local embedding model, stores
the vectors in this repo's vector database, then retrieves relevant chunks for
a local LLM answer.

Current CLI commands:

```sh
laptop-ai init
laptop-ai index <folder>
laptop-ai ask <question>
laptop-ai stats
```

## Why Local-First?

Your notes may contain private thoughts, drafts, plans, or credentials. Laptop AI
defaults to local-only behavior:

- embeddings use local Ollama
- answers use local Ollama
- config defaults to `local_only: true`
- cloud models are disabled by default
- secret-looking files and content are skipped before indexing
- audit logs store metadata only, not chunk text or questions

## Quickstart

Install and start Ollama, then pull the models used by the demo:

```sh
ollama serve
ollama pull nomic-embed-text
ollama pull llama3
```

Build the CLI:

```sh
go build ./cmd/laptop-ai
```

Run the demo notes:

```sh
./laptop-ai init
./laptop-ai index ./examples/notes
./laptop-ai ask "what controls movement in my biology notes?"
```

On Windows PowerShell, use:

```powershell
.\laptop-ai.exe init
.\laptop-ai.exe index .\examples\notes
.\laptop-ai.exe ask "what controls movement in my biology notes?"
```

## Requirements

- Go 1.22 or newer
- Ollama running locally at `http://localhost:11434`
- Ollama model `nomic-embed-text`
- Ollama model `llama3`

The `ollama` command does not have to be on PATH after the server is running,
but the HTTP server must be reachable.

Example output:


The wording may vary because the answer is generated by a local LLM. The
important behavior is that the answer is based on the retrieved notes and the
sources section includes the source file.

## Architecture

```text
folder
  -> security checks and ignore rules
  -> text extraction
  -> chunking
  -> local embeddings through Ollama
  -> custom vector DB
  -> cosine search
  -> local LLM answer through Ollama
  -> answer + sources
```

Main packages:

- `internal/cli`: Cobra commands and user flow
- `internal/indexer`: folder scanning, metadata, change detection
- `internal/extractor`: text, markdown, HTML, PDF, DOCX extraction
- `internal/chunker`: overlapping word chunks with source metadata
- `internal/embeddings`: local Ollama embeddings
- `internal/vectordb`: WAL, memtable, segment files, cosine search
- `internal/llm`: local Ollama answer generation and prompt defense
- `internal/security`: denylist, allowlist, secret detection, cloud guards
- `internal/audit`: metadata-only audit log

## Security Model

Laptop AI is designed to fail closed for common local privacy risks:

- refuses whole-home indexing
- only indexes explicitly provided folders
- skips denylisted paths such as `.env`, private keys, SSH folders, and
  dependency folders
- loads `.laptopaiignore` from the indexed folder
- skips files that contain secret-looking values
- skips symlinks by default
- wraps retrieved context as untrusted document data in the LLM prompt
- logs query source paths, not question text or answer text

Cloud-provider preview/redaction helpers exist, but the CLI does not currently
send notes to a cloud model.

## Current Limitations

- `ask` depends on a running local Ollama server.
- Local LLM generation can be slow on CPU-only machines.
- Re-indexing changed files appends new chunk records; there is no deletion or
  compaction command yet.
- Source output follows the model's `[1]` citations when present, but it is not
  a full citation verifier.
- Encrypted storage helpers exist, but vector DB files are not encrypted yet.
- Segment indexes are full ID-to-offset maps, not true sparse/block indexes.
- There is no `search`, `sources`, `forget`, or `doctor` CLI command yet.
- Cloud adapter code is not wired into the CLI.

## Roadmap
